# REQ-01 — เอกสารความต้องการ: กระบวนการแจ้งเรื่องร้องทุกข์ (Complaint Intake)

> **ระบบ:** ระบบรับเรื่องร้องเรียนศูนย์ดำรงธรรมจังหวัดศรีสะเกษ (DCMS)
> **ขอบเขตเอกสาร:** ความต้องการของ **กระบวนการแจ้ง/บันทึกเรื่องร้องทุกข์** (ฟอร์มยื่นเรื่องและการบันทึกของเจ้าหน้าที่) — สรุปจากการวิเคราะห์ร่วมกับผู้ใช้
> **สถานะ:** Requirement (อนุมัติแล้ว นำไปปรับปรุงเอกสาร Planning `01`–`10`)
> **วันที่:** 2026-06-19

---

## 1. วัตถุประสงค์ (Purpose)

กำหนดข้อมูลและกติกาที่ระบบต้องรองรับในการ **รับเรื่องร้องทุกข์/ร้องเรียน** ให้ครบถ้วนตามแนวทางปฏิบัติของศูนย์ดำรงธรรม ครอบคลุมทั้ง:

- การยื่นเรื่องโดย **ประชาชน** (แบบสมาชิก และแบบไม่สมาชิก/Guest) ผ่านหน้าเว็บสาธารณะ
- การ **บันทึก/แจ้งเรื่องแทนประชาชนโดยเจ้าหน้าที่ศูนย์ดำรงธรรม** จากหลายช่องทาง

เอกสารนี้เป็นความต้องการระดับธุรกิจ/ฟังก์ชัน ใช้อ้างอิงร่วมกับเอกสารออกแบบใน `docs/planning/`

### 4.1 ข้อมูลส่วนตัวผู้ร้อง

| RQ-1.2 | **บังคับกรอกเบอร์โทรศัพท์ผู้ร้องเสมอ** ทั้งสองกรณี (ออกนาม/ปกปิดตัวตน) เพื่อให้ติดต่อกลับได้ | High |

## 8. ข้อกำหนดด้านส่วนติดต่อผู้ใช้ (UI)

- ฟอร์มยื่นเรื่อง (Public / Citizen / Staff) ต้องมีฟิลด์ตามข้อ 5 ครบถ้วน
- ส่วนแผนที่ใช้คอมโพเนนต์ `LocationMapPicker` (Leaflet) — มีปุ่ม "ใช้ตำแหน่งปัจจุบัน (GPS)" และคลิกเลือกพิกัดบนแผนที่
- Checkbox "ปกปิดตัวตน" พร้อมข้อความอธิบายว่าข้อมูลยังถูกเก็บเพื่อติดต่อกลับแต่ไม่แสดงต่อเจ้าหน้าที่ และเบอร์โทรยังบังคับกรอก
- หน้าฟอร์มของเจ้าหน้าที่ต้องมีตัวเลือกช่องทางการรับเรื่องครบ 9 ช่องทาง

## 9. เกณฑ์การยอมรับ (Acceptance Criteria)

> จัดกลุ่มตามความต้องการ (RQ) แต่ละข้อ — แต่ละเกณฑ์ทดสอบได้และให้ผล **ผ่าน/ไม่ผ่าน** ชัดเจน
> สัญลักษณ์: ✅ = เคสที่ต้องทำได้สำเร็จ (positive) · ⛔ = เคสที่ระบบต้องปฏิเสธ/ป้องกัน (negative)

### AC-1 ข้อมูลส่วนตัวผู้ร้อง (RQ-1)

- [ ] ⛔ ยื่นเรื่องโดยเว้นช่องเบอร์โทรว่าง → ระบบ **ไม่บันทึก** และแสดงข้อความเตือน (ทั้งกรณีออกนามและปกปิดตัวตน)
- [ ] ✅ ยื่นเรื่อง **แบบออกนาม** → officer/chief/admin/super_admin เห็นข้อมูลผู้ร้อง (ชื่อ/เบอร์/ที่อยู่/เลขบัตร) ตามสิทธิ์
- [ ] ✅ ยื่นเรื่อง **แบบปกปิดตัวตน** → เจ้าหน้าที่ทุก Role (รวม officer/chief/admin) **ไม่เห็น** ข้อมูลผู้ร้อง + `citizen_id` แต่ข้อมูลยังถูกบันทึกครบใน DB
- [ ] ✅ super_admin กด "เปิดเผยตัวตน" เรื่องปกปิดตัวตน + กรอกเหตุผล → เห็นข้อมูลผู้ร้อง และระบบบันทึก `anonymous_reveal_logs` + `audit_logs`
- [ ] ⛔ super_admin กด "เปิดเผยตัวตน" โดยไม่กรอกเหตุผล → ระบบไม่อนุญาต
- [ ] ⛔ Role อื่นที่ไม่ใช่ super_admin → ไม่เห็นปุ่ม/เรียก API เปิดเผยตัวตนไม่ได้ (403)

### AC-2 การระบุเรื่องร้องเรียน (RQ-2)

- [ ] ⛔ เว้นว่างฟิลด์บังคับใด ๆ (ชื่อเรื่อง, รายละเอียด, ประเภทงานบริการ, ลักษณะเรื่อง, ประเภทเรื่อง, ประเภทผู้ร้องเรียน, ช่องทาง) → ระบบไม่บันทึกและเตือนรายฟิลด์
- [ ] ✅ เลือก **ประเภทงานบริการ / ลักษณะเรื่อง / ประเภทผู้ร้องเรียน** ได้จากรายการ Master Data ที่ active
- [ ] ✅ super_admin/admin เพิ่ม-แก้-ปิดใช้งานรายการ Master Data ทั้ง 3 ชุด + จังหวัด ผ่านหน้า Settings แล้วรายการอัปเดตในฟอร์มยื่นเรื่อง
- [ ] ✅ รายการที่ถูกปิดใช้งาน (is_active=false) ไม่ปรากฏให้เลือกในฟอร์มยื่นเรื่องใหม่

### AC-3 การแนบไฟล์หลักฐาน (RQ-3)

- [ ] ✅ แนบไฟล์ได้ **หลายไฟล์** ในการยื่นครั้งเดียว ทั้งฝั่งเจ้าหน้าที่และประชาชน (Guest/สมาชิก)
- [ ] ⛔ แนบไฟล์ขนาด > 10 MB → ระบบปฏิเสธพร้อมแจ้งเตือน
- [ ] ⛔ แนบไฟล์นามสกุลนอกเหนือ jpg/jpeg/png/pdf/doc/docx → ระบบปฏิเสธพร้อมแจ้งเตือน
- [ ] ✅ ยื่นเรื่องได้สำเร็จแม้ **ไม่แนบไฟล์** (ไฟล์แนบเป็น optional)

### AC-4 จุดเกิดเหตุ (RQ-4)

- [ ] ✅ ระบุจังหวัด/อำเภอ/ตำบล/รหัสไปรษณีย์/สถานที่เกิดเหตุ ได้ครบ และบันทึกถูกต้อง
- [ ] ✅ Dropdown แบบลำดับชั้น: เลือกจังหวัด → อำเภอกรองตามจังหวัด → ตำบลกรองตามอำเภอ
- [ ] ✅ จังหวัดมีค่าเริ่มต้นเป็น "ศรีสะเกษ"
- [ ] ⛔ กรอกรหัสไปรษณีย์ไม่ใช่ 5 หลัก → ระบบเตือน (ถ้ากรอก)
- [ ] ✅ ยื่นเรื่องได้สำเร็จแม้เว้นข้อมูลจุดเกิดเหตุ (ทั้งกลุ่มเป็น optional)

### AC-5 การกำหนดจุดบนแผนที่ (RQ-5)

- [ ] ✅ กดปุ่ม "ใช้ตำแหน่งปัจจุบัน (GPS)" → แผนที่ปักหมุดและระบบเก็บ latitude/longitude
- [ ] ✅ คลิกเลือกพิกัดเองบนแผนที่ Leaflet → ระบบเก็บ latitude/longitude ของจุดที่เลือก
- [ ] ✅ ยื่นเรื่องได้สำเร็จแม้ **ไม่ปักหมุด** (พิกัดเป็น optional)

### AC-6 การแจ้งเรื่องโดยเจ้าหน้าที่ + ช่องทาง (RQ-6)

- [ ] ✅ เจ้าหน้าที่ (officer/chief) บันทึกเรื่องแทนประชาชนได้ → เรื่องถูกบันทึกด้วย `source = STAFF`
- [ ] ✅ เจ้าหน้าที่เลือกช่องทางการรับเรื่องได้ครบทั้ง 9 ช่องทาง (จดหมาย, ลงพื้นที่/หน่วยเคลื่อนที่เร็ว, ตู้ราชสีห์, 1567, Walk In, เว็บไซต์, MOI1567, ผ่านหน่วยงานอื่น, อื่น ๆ)

### AC-7 ช่องทางการยื่นและพฤติกรรมร่วม (Member / Guest / Staff)

- [ ] ✅ **สมาชิก (Citizen)**: ฟอร์ม pre-fill ข้อมูลผู้ร้องจากโปรไฟล์ (แก้ก่อนส่งได้) · เรื่องผูก `citizen_id` · เห็นในหน้า "เรื่องของฉัน"
- [ ] ✅ **Guest**: ยื่นได้โดยไม่ Login · `citizen_id = NULL` · ติดตามสถานะด้วยเลขที่เรื่อง
- [ ] ✅ ทุกช่องทางสร้างเรื่องที่สถานะเริ่มต้น **NEW** และได้เลขที่เรื่องรูปแบบ **DC-YYYYMM-XXXX**
- [ ] ✅ ระบบเขียน **audit log** ทุกครั้งที่สร้างเรื่อง
- [ ] ✅ **Snapshot (CKD-5)**: เมื่อสมาชิกแก้โปรไฟล์ภายหลัง ข้อมูลผู้ร้องในเรื่องที่ยื่นไปแล้ว **ไม่เปลี่ยนแปลง**
