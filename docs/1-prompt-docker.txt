================================================================================
  ⚠️ บทเรียนจากปัญหาที่เจอจริง (Lessons Learned)
================================================================================

1. Port 5000 ชนกับ macOS AirPlay Receiver
   - อาการ: "bind: address already in use" เมื่อ docker compose up
   - แก้ไข: เปลี่ยนเป็น port 5001 (หรือ port อื่นที่ไม่ชน)
   - ต้องแก้ทั้ง: docker-compose.yml, Dockerfile.dev, server.js, vite.config.js

2. phpMyAdmin ไม่รองรับ Apple Silicon (ARM) โดยตรง
   - อาการ: "platform does not match" warning, container อาจไม่ทำงาน
   - แก้ไข: เพิ่ม platform: linux/amd64 ให้กับ phpMyAdmin service
   - Docker Desktop จะใช้ Rosetta 2 emulation อัตโนมัติ

3. version: "3.8" ล้าสมัย
   - อาการ: "the attribute 'version' is obsolete" warning
   - แก้ไข: ลบ version ออกจาก docker-compose.yml

4. node_modules ใน Docker ต้องจัดการด้วย anonymous volume
   - อาการ: npm packages หายหรือ architecture ไม่ตรง
   - แก้ไข: ใช้ /app/node_modules เป็น anonymous volume ใน docker-compose.yml
   - สำคัญ: เมื่อเพิ่ม npm package ใหม่ ต้อง rebuild container

5. MySQL ต้องมี healthcheck + depends_on condition
   - อาการ: Backend/phpMyAdmin เริ่มก่อน MySQL พร้อม → connection refused
   - แก้ไข: ตั้ง healthcheck ด้วย mysqladmin ping + depends_on condition: service_healthy
