## หมวด A — รับเรื่องร้องเรียน

### A-01: สร้างเรื่องร้องเรียนใหม่

**ผู้ทดสอบ**: officer / admin
**ขั้นตอน**:
1. Login ด้วย officer account
2. ไปที่ เมนู "รายการเรื่องร้องเรียน" → คลิก "รับเรื่องใหม่"
3. กรอกข้อมูลครบทุกช่องที่จำเป็น
4. กด "บันทึก"

**ผลลัพธ์ที่คาดหวัง**:
- [ ] สร้างเรื่องร้องเรียนสำเร็จ
- [ ] เลข complaint_number ถูกสร้างอัตโนมัติ (รูปแบบ DD-YYYYMMDD-NNNNN)
- [ ] สถานะ = NEW
- [ ] ปรากฏในรายการเรื่องร้องเรียน

### A-02: สร้างเรื่องแบบไม่เปิดเผยตัวตน

**ผู้ทดสอบ**: officer
**ขั้นตอน**: เช่นเดียวกับ A-01 แต่ติ๊ก "ไม่ระบุตัวตน"
**ผลลัพธ์ที่คาดหวัง**:
- [ ] บันทึกสำเร็จ
- [ ] ในหน้ารายละเอียด ข้อมูลผู้ร้องถูกซ่อน (แสดง "ไม่เปิดเผย")

## หมวด B — กระบวนการมอบหมาย

### B-01: รับเรื่องและมอบหมายหน่วยงาน

**ผู้ทดสอบ**: officer / admin
**ขั้นตอน**:
1. เลือกเรื่องที่สถานะ NEW
2. คลิก "รับเรื่อง" (T-01: NEW → RECEIVED)
3. เลือกหน่วยงานที่จะมอบหมาย
4. คลิก "มอบหมาย" (T-03: RECEIVED → ASSIGNED)

**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น ASSIGNED
- [ ] หน่วยงานที่มอบหมายปรากฏ
- [ ] Timeline แสดงการเปลี่ยนสถานะ

### B-02: ส่งต่อเรื่อง (Forward)

**ผู้ทดสอบ**: officer / admin
**ขั้นตอน**: เรื่องที่ ASSIGNED → ส่งต่อไปหน่วยงานอื่น (T-10)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] หน่วยงานเดิมถูก deactivate
- [ ] หน่วยงานใหม่ถูก assign
- [ ] Timeline แสดง "ส่งต่อ"

### B-03: คืนเรื่อง (Return)

**ผู้ทดสอบ**: agency_officer
**ขั้นตอน**: เรื่องที่ ASSIGNED → คลิก "คืนเรื่อง" พร้อมระบุ return_reason (T-07)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น RETURNED
- [ ] return_reason บันทึกและแสดงใน timeline

---

## หมวด C — ดำเนินการแก้ไข

### C-01: รับเรื่องและดำเนินการ (In Progress)

**ผู้ทดสอบ**: agency_officer
**ขั้นตอน**:
1. Login ด้วย agency_officer
2. เลือกเรื่องที่ถูก assign ให้
3. คลิก "รับดำเนินการ" (T-05: ASSIGNED → IN_PROGRESS)

**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น IN_PROGRESS
- [ ] due_date ปรากฏ

### C-02: แจ้งความคืบหน้า

**ผู้ทดสอบ**: agency_officer
**ขั้นตอน**: เรื่อง IN_PROGRESS → เพิ่ม "ความคืบหน้า" ด้วย content (T-08)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] Update บันทึกใน timeline
- [ ] เนื้อหาแสดงถูกต้อง

### C-03: รายงานผลการดำเนินการ (Resolve)

**ผู้ทดสอบ**: agency_officer
**ขั้นตอน**: เรื่อง IN_PROGRESS → คลิก "รายงานผล" พร้อม content (T-09)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น RESOLVED
- [ ] เนื้อหา resolution แสดงใน timeline

---

## หมวด D — ปิดเรื่อง / ปฏิเสธ

### D-01: ปิดเรื่อง (Close)

**ผู้ทดสอบ**: officer / admin
**ขั้นตอน**: เรื่อง RESOLVED → คลิก "ปิดเรื่อง" พร้อม closed_summary (T-11)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น CLOSED
- [ ] closed_at และ closed_summary บันทึก
- [ ] ปุ่ม "แก้ไข" ถูกซ่อน (ไม่สามารถแก้เรื่อง CLOSED ได้)

### D-02: ปฏิเสธเรื่อง (Reject)

**ผู้ทดสอบ**: officer / admin
**ขั้นตอน**: เรื่อง NEW/RECEIVED → คลิก "ปฏิเสธ" พร้อมระบุเหตุผล (T-02)
**ผลลัพธ์ที่คาดหวัง**:
- [ ] สถานะเปลี่ยนเป็น REJECTED
- [ ] rejection_reason บันทึก

---

## หมวด E — Dashboard & รายงาน

### E-01: Dashboard Summary

**ผู้ทดสอบ**: admin, officer, executive
**ขั้นตอน**: Login → ดูหน้า Dashboard
**ผลลัพธ์ที่คาดหวัง**:
- [ ] จำนวนเรื่องแยกตามสถานะแสดงถูกต้อง
- [ ] จำนวนเรื่อง overdue แสดง
- [ ] กราฟ/สถิติโหลดได้ (ไม่ error)

### E-02: Export Excel

**ผู้ทดสอบ**: admin
**ขั้นตอน**: ไปที่ รายงาน → Export Excel
**ผลลัพธ์ที่คาดหวัง**:
- [ ] ดาวน์โหลดไฟล์ .xlsx ได้
- [ ] ข้อมูลในไฟล์ตรงกับหน้าจอ
- [ ] ตัวอักษรภาษาไทยแสดงถูกต้อง

## หมวด F — Public & Citizen

### F-01: Public Complaint Tracking

**ผู้ทดสอบ**: ประชาชน (ไม่ต้อง login)
**ขั้นตอน**: เปิด URL `/track` → ใส่เลขที่เรื่องร้องเรียน
**ผลลัพธ์ที่คาดหวัง**:
- [ ] เห็นสถานะและความคืบหน้า
- [ ] ไม่เห็นข้อมูลส่วนตัวผู้ร้อง

### F-02: Citizen Self-Register & Submit

**ผู้ทดสอบ**: ประชาชน
**ขั้นตอน**: สมัครบัญชี citizen → Login → ยื่นเรื่องร้องเรียน
**ผลลัพธ์ที่คาดหวัง**:
- [ ] สมัครบัญชีสำเร็จ
- [ ] Login ได้
- [ ] ยื่นเรื่องสำเร็จ
- [ ] เห็นเรื่องของตัวเองในประวัติ

## หมวด G — Security

### G-01: ไม่สามารถเข้าถึง API โดยไม่มี Token

```bash
curl https://<domain>/api/complaints
# Expected: 401 Unauthorized
```

### G-02: Role ที่ไม่มีสิทธิ์เข้าถึง endpoint ไม่ได้

**ขั้นตอน**: Login ด้วย agency_officer → พยายามเรียก admin-only endpoint
**ผลลัพธ์ที่คาดหวัง**: 403 Forbidden
